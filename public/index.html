    📜 Conversation History

    Every conversation is recorded to local SQLite. Click any past session to restore it — the SDK resumes from where you left off, your messages reload into the chat, and replies continue the same thread.

    ⚙️ Settings

    Per-integration tokens and URLs. Values saved here override any matching environment variable on the next request. Secrets are never returned to the browser — you see a masked preview, and leaving a secret field blank keeps the existing value.

    🧠 Persistent Memory

    Memories are injected into the agent's system prompt on every message. Use them for facts, preferences, and context you want to survive restarts.

      📌 Context Pins

      Pins are injected into the selected agent's system prompt on every turn. File pins are re-read from disk each turn — edit the file and the agent sees the latest, no re-save needed. Snippet pins are fixed text. Great for a writing-style doc, a project spec, or a glossary.

        🔌 MCP Servers

        Connect Model Context Protocol servers to an agent — their tools light up automatically when that agent runs. stdio spawns a local process; http/sse connect to a URL. Use Test to verify a server connects before relying on it.

          🧩 Skills

          Agent Skills discovered in …/.claude/skills and ~/.claude/skills. Enable a skill for an agent and it loads into that agent's system prompt on every turn. Skills you install here live in your user skills folder and can be removed with 🗑.

            Add a skill three ways. Everything you add installs to ~/.claude/skills as a standard SKILL.md — no external registry, no off-device upload. Pasted skills are scanned for risky patterns before install.

                When an agent completes a useful multi-step task, the 💡 nudge under its reply distills it into a draft skill that lands here for review. A proposal is distilled from a transcript that may include content the agent read from outside, so it's treated as untrusted: installing runs the same safety scan, and a high-severity draft is gated.

                  🌐 Browser Automation

                  Let an agent drive a real browser (powered by the Playwright MCP server) to navigate, click, type, and read pages. Every navigation passes a permission gate. A hard deny-list always blocks localhost, private network addresses, and non-web protocols, even in Open mode, so a wandering or prompt-injected agent can never reach your machine or LAN.

                  This agent can visit only the domains you allow below. Subdomains are included.
                    🎭 Agent Personality

                    Give an agent a voice. Pick a ready-made tone, or open the Soul Builder to craft a custom one. A personality only adds voice and context on top of the agent's own role — it never changes its name or job. Privacy, boundary, and continuity rules are locked: you can make an agent friendlier, but you can't talk it out of refusing prompt injection or leaking its system prompt.

                    These add to — never replace — the locked privacy & boundary rules.

                    🕒 Scheduled Agent Runs

                    Schedules wake an agent on a cron expression. Each fire enqueues a task with fresh context (no session resume) and runs through CostGuard preflight. If the OAuth session dies, the schedule auto-pauses instead of looping errors. After 3 consecutive non-OAuth errors it auto-pauses too.
